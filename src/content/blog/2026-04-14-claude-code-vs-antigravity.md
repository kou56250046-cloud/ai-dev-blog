## そもそも Claude Code / Antigravity とは？

### Claude Code

Claude Code は、Anthropic が開発した **AI コーディングツール**です。ターミナル（CLI）を中心に、VS Code・JetBrains・デスクトップアプリ・ブラウザなど幅広い環境で動作します。

最大の特徴は **ターミナルファースト** のアプローチ。開発者がコマンドラインから自然言語で指示を出し、Claude がコードベース全体を理解した上でファイルの読み書き・コマンド実行・git 操作まで一気通貫で行います。

```bash
# ターミナルから起動するだけ
claude

# 自然言語で指示
> src/utils/date.ts に日本語対応の日付フォーマット関数を追加して
```

### Google Antigravity

Antigravity は、Google が **2025年11月に発表した AI 搭載の統合開発環境（IDE）**です。VS Code をベースにしたスタンドアロンアプリで、Windows・macOS・Linux に対応しています。

最大の特徴は **エージェントファースト** のアプローチ。タスクを入力すると、AI エージェントが計画を立て、コードを書き、テストを実行し、検証済みの結果を返してくれます。

## 手順

### 1. アーキテクチャの違いを理解する

両ツールの設計思想は根本的に異なります。

| 観点 | Claude Code | Antigravity |
|---|---|---|
| 設計思想 | ターミナルファースト | エージェントファースト |
| 人間の関与 | 人間が逐次承認・操作 | エージェントが自律的に実行 |
| ベース | 独自 CLI + エディター拡張 | VS Code フォーク（スタンドアロン IDE）|
| AI モデル | Claude Opus 4.6 / Sonnet 4.6 | Gemini 3 Pro / Claude Sonnet 4.6 / GPT-OSS |

**Claude Code** は「人間がハンドルを握り、AI がナビゲーションする」イメージです。ファイル操作やコマンド実行の前に承認を求めてくるため、開発者が常にコントロールを保てます。

**Antigravity** は「AI がドライバーで、人間はマネージャー」というイメージ。タスクを投げたら、エージェントが 計画 → 実装 → テスト → 検証 を自律的に進めます。

### 2. 対応環境を比較する

| 環境 | Claude Code | Antigravity |
|---|---|---|
| ターミナル（CLI）| フル対応 | エージェント経由で利用 |
| VS Code | 拡張機能で対応 | ベース IDE そのもの |
| JetBrains | プラグインで対応 | **非対応** |
| デスクトップアプリ | 専用アプリあり | IDE 自体がアプリ |
| ブラウザ | claude.ai/code で対応 | **非対応**（ローカルのみ）|
| モバイル | iOS アプリ経由で操作可 | **非対応** |

> **ポイント**: Claude Code は「好きな環境で使える」柔軟さが魅力。Antigravity は IDE そのものなのでツール切り替えが不要ですが、JetBrains ユーザーやリモート作業には向きません。

### 3. エージェント機能を比較する

#### Claude Code のエージェント機能

- `/compact` でコンテキストを圧縮しながらセッションを管理
- 複数のサブエージェントを生成して**並列処理**
- **Agent SDK** で完全カスタムのエージェントを構築可能
- **MCP（Model Context Protocol）** で外部ツール（Slack・Jira・Google Drive など）と連携
- **CLAUDE.md** でプロジェクト固有のルールを設定

#### Antigravity のエージェント機能

- **Gravity Mode**: タスクを入力すると AI が計画を立て自律実行
- **Manager Surface**: 複数エージェントを並列で監視・操作するダッシュボード
- **ブラウザ統合**: AI が自動的にコードをテスト実行
- **Rules / Workflows**: エージェントのルール設定とカスタムワークフロー
- **画像生成機能**を内蔵し、必要なアセットを自動生成

> **ポイント**: Claude Code は開発者が細かくコントロールしたい場合に強く、Antigravity はタスクを丸投げして自律実行させたい場合に強い。

### 4. 使えるAIモデルを確認する

| モデル | Claude Code | Antigravity |
|---|---|---|
| Claude Opus 4.6 | ✅ デフォルト使用可能 | ❌ 非対応 |
| Claude Sonnet 4.6 | ✅ デフォルト使用可能 | ✅ 対応 |
| Gemini 3 Pro | ❌ 非対応 | ✅ デフォルト使用可能 |
| Gemini 3.1 Flash | ❌ 非対応 | ✅ Free プランで使用可能 |
| GPT-OSS | ❌ 非対応 | ✅ 対応 |

Claude Code は Anthropic 製モデルに特化しており、特に **Opus 4.6（最高性能）を使えるのが最大のアドバンテージ**です。Antigravity は複数ベンダーのモデルをサポートしているため、タスクに応じて最適なモデルを選べる柔軟さがあります。ただし、Opus 4.6 は使えません。

### 5. 料金プランを比較する

#### Claude Code の料金

Claude Code は最低でも Pro プラン（月額 $20・約¥3,120）が必要です。

| プラン | 月額 | Claude Code の利用 |
|---|---|---|
| Free | 無料 | **利用不可** |
| Pro | $20（約¥3,120）| 利用可能（標準的な使用量）|
| Max 5x | $100（約¥15,600）| 利用可能（Pro の5倍の使用量）|
| Max 20x | $200（約¥31,200）| 利用可能（Pro の20倍・フル活用向け）|
| Team | $25〜$150/席 | 利用可能（チーム開発向け）|
| Enterprise | 要問い合わせ | フル機能 |

#### Antigravity の料金

現在はパブリックプレビュー中で、個人利用は**無料**です。

| プラン | 月額 | 内容 |
|---|---|---|
| Free | **無料** | Gemini 3.1 Flash・AIコード提案は1日50回まで |
| Pro | $25（約¥3,900）| Gemini 3.1 Pro / 無制限補完 |
| Enterprise | $45/席（約¥7,020）| プライベートモデル / SOC 2 対応 |

> **ポイント**: 初期コストは Antigravity が圧倒的に有利。ただし Claude Code の Max プランでは最高性能の Opus 4.6 が使い放題のため、コード品質・生産性を最重視するプロ開発者にはコストに見合う価値があります。

## メリット・デメリット

### Claude Code のメリット

- **最高品質のコード生成**: Opus 4.6 による高精度な出力
- **環境の柔軟性**: CLI・VS Code・JetBrains・ブラウザ・モバイルと幅広く対応
- **開発者がコントロールを保てる**: 操作の承認フローで安全性が高い
- **豊富なカスタマイズ**: CLAUDE.md・MCP・フック・スラッシュコマンド
- **セキュリティ認証**: SOC 2 Type II・ISO 27001 取得済み

### Claude Code のデメリット

- 無料プランでは使えない（最低 $20/月）
- ヘビーユースはコストが高い（Max プランは $100〜$200/月）
- ターミナル操作に慣れていない初心者にはハードルがある

### Antigravity のメリット

- **無料で始められる**: パブリックプレビュー中は個人無料
- **エージェントファースト**: タスクを投げるだけで AI が自律的に実行
- **複数モデル対応**: Gemini・Claude・GPT-OSS から選択可能
- **Manager Surface**: 複数エージェントの並列管理が直感的
- **画像生成内蔵**: UI 開発時にアセットも自動生成

### Antigravity のデメリット

- **まだプレビュー段階**: バグやフリーズの報告が多い
- 拡張機能の互換性が不完全（VS Code の拡張が全て動くわけではない）
- バッテリー・CPU 消費が大きい
- **自律性のリスク**: エージェントが意図しない操作を行う事例が報告されている
- JetBrains・モバイル非対応
- プロダクション実績が少ない（2025年11月リリース）

## 用語解説

| 用語 | 意味 |
|---|---|
| CLI（コマンドラインインターフェース）| ターミナル（黒い画面）でコマンドを入力して操作する方式 |
| エージェント | 指示を受けて自律的にタスクを実行する AI プログラム |
| MCP（Model Context Protocol）| AI ツールと外部サービスを繋ぐ標準プロトコル |
| Gravity Mode | Antigravity の自律実行モード。タスクを入力すると AI が計画〜実行まで一貫して行う |
| Manager Surface | Antigravity の複数エージェント管理ダッシュボード |
| SOC 2 Type II | セキュリティ管理体制の第三者認定。企業の情報セキュリティ対策が適切かを証明する |
| ISO 27001 | 情報セキュリティマネジメントに関する国際認定規格 |

## どんな人にどちらがおすすめ？

### 初心者・プログラミング学習中の方 → **Antigravity**

- 無料で始められる
- GUI 中心でターミナルに慣れていなくても使いやすい
- AI が計画から実行まで自動で進めてくれる

ただし、まだプレビュー版のため不安定な場面があります。安定性を求める場合は Claude Code の Pro プランも検討してみてください。

### 中級者・業務エンジニアの方 → **用途に応じて使い分け**

| 用途 | おすすめ |
|---|---|
| 本番コード・品質重視の開発 | Claude Code |
| プロトタイプ・実験的なプロジェクト | Antigravity |
| JetBrains ユーザー | Claude Code（一択）|
| Google Cloud との連携 | Antigravity |
| セキュリティ要件が厳しい企業 | Claude Code |

### 両方を使い分けるのが最強

実は、**両方を使い分けるのが最も賢い選択**です：

- **重要な本番コード** → Claude Code（高品質・安全性重視）
- **プロトタイプ・実験** → Antigravity（スピード・コスト重視）
- **Antigravity で Claude Sonnet 4.6 を選べば**、両ツールの良いとこ取りも可能
